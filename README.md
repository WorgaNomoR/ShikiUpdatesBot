# 🎌 [ShikiUpdatesBot](https://github.com/WorgaNomoR/ShikiUpdatesBot) — Shikimori Activity Tracker

Telegram-бот, который следит за аниме и мангой любого пользователя на [Shikimori](https://shikimori.io) и отправляет весёлые уведомления подписчикам.
Достаточно указать ник отслеживаемого пользователя и своё отображаемое имя в настройках — бот готов к работе.

## Что умеет

- 📡 Проверяет историю пользователя каждые **15 минут**
- 🎬 Реагирует на события: добавил в список, начал смотреть/читать, пересматривает, бросил, завершил, поставил оценку, изменил оценку
- ✅ Поддерживает **аниме** (TV, фильм, OVA, ONA) и **мангу** (кроме ваншотов)
- 👥 Система подписки — каждый подписывается сам командой `/start`
- 📋 Команда `/status` показывает что сейчас смотрит и читает пользователь

## Команды

| Команда | Описание |
|---|---|
| `/start` | Подписаться на уведомления |
| `/stop` | Отписаться |
| `/status` | Что сейчас смотрит и читает отслеживаемый пользователь |
| `/subs` | Список подписчиков *(только для владельца)* |

## Установка

### 1. Клонировать репозиторий

```bash
git clone https://github.com/WorgaNomoR/ShikiUpdatesBot.git
cd ShikiUpdatesBot
```

### 2. Установить зависимости

```bash
pip install -r requirements.txt
```

### 3. Создать бота

Написать [@BotFather](https://t.me/BotFather) → `/newbot` → получить токен.

### 4. Узнать свой Telegram ID

Написать [@userinfobot](https://t.me/userinfobot) — он пришлёт ваш числовой ID.

### 5. Задать переменные окружения

```bash
export BOT_TOKEN="токен_от_BotFather"
export OWNER_ID="ваш_telegram_id"
```

Или создать файл `.env` (см. `.env.example`) и загрузить через `python-dotenv`.

### 6. Запустить

```bash
python main.py
```

## Конфигурация

Основные параметры в начале `main.py`:

| Константа | По умолчанию | Описание |
|---|---|---|
| `SHIKI_USER` | `WNR` | Ник на Shikimori |
| `DISPLAY_NAME` | `Ворга` | Отображаемое имя в сообщениях |
| `SHIKI_BASE_URL` | `https://shikimori.io` | Домен — менять при смене зеркала |
| `CHECK_INTERVAL` | `900` (15 мин) | Интервал проверки в секундах |

## Файлы данных

Бот хранит два файла с состоянием — **не коммитить**:

- `seen_ids.json` — ID уже обработанных событий истории
- `subscribers.json` — список подписчиков

По умолчанию файлы создаются в текущей рабочей директории. Если нужно хранить их в другом месте (например, в папке постоянного хранилища на хостинге), укажите путь через переменную окружения:

```
DATA_DIR=/путь/к/папке
```

При первом запуске бот молча запоминает текущую историю и не спамит прошлыми событиями — отслеживает только новые.
