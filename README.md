Telegram-бот, который следит за аниме и мангой любого пользователя на [Shikimori](https://shikimori.io) и отправляет весёлые уведомления подписчикам.
Достаточно указать ник отслеживаемого пользователя и своё отображаемое имя в настройках — бот готов к работе.

## Что умеет

- 📡 Проверяет историю и избранное пользователя каждые **15 минут**
- 🎬 Реагирует на события: добавил в список, начал смотреть/читать, пересматривает/перечитывает, бросил, завершил, поставил оценку, изменил оценку
- ⭐ Отслеживает новые добавления в избранное: аниме, мангу, персонажей и людей индустрии
- ✅ Поддерживает **аниме** (TV, фильм, OVA, ONA) и **мангу** (кроме ваншотов и додзинси)
- 👥 Система подписки — каждый подписывается сам командой `/start`
- 📋 Команда `/status` показывает что сейчас смотрит и читает пользователь
- 📢 Команда `/broadcast` позволяет владельцу написать подписчикам напрямую
- 💾 Команды `/export` и `/import` для переноса списка подписчиков между хостингами
- ⚠️ При ошибках фоновой проверки пишет в лог и уведомляет владельца в Telegram с антиспамом

## Команды

| Команда | Описание |
|---|---|
| `/start` | Подписаться на уведомления |
| `/stop` | Отписаться |
| `/status` | Что сейчас смотрит и читает отслеживаемый пользователь |
| `/broadcast` | Написать сообщение подписчикам *(только для владельца)* |
| `/cancel` | Отменить текущую операцию *(только для владельца)* |
| `/subs` | Список подписчиков *(только для владельца)* |
| `/export` | Скачать `subscribers.json` *(только для владельца)* |
| `/import` | Загрузить `subscribers.json` из файла *(только для владельца)* |

## Установка

### 1. Клонировать репозиторий

```bash
git clone https://github.com/WorgaNomoR/ShikiUpdatesBot.git
cd ShikiUpdatesBot
```

### 2. Установить зависимости

```bash
pip install -r requirements.txt
```

### 3. Создать бота

Напиши [@BotFather](https://t.me/BotFather) → `/newbot` → получи токен.

### 4. Узнать свой Telegram ID

Напиши [@userinfobot](https://t.me/userinfobot) — он пришлёт твой числовой ID.

### 5. Задать переменные окружения

```bash
export BOT_TOKEN="токен_от_BotFather"
export OWNER_ID="твой_telegram_id"
```

Или создай файл `.env` (см. `.env.example`) и загрузи его через `python-dotenv`.

### 6. Запустить

```bash
python main.py
```

## Запуск через Docker

Альтернативный способ запуска — через Docker-контейнер. Это упрощает развёртывание и гарантирует одинаковую среду выполнения на любом сервере.

### 1. Установить Docker

Убедись, что у тебя установлен [Docker](https://docs.docker.com/get-docker/) и [Docker Compose](https://docs.docker.com/compose/install/).

### 2. Создать файл `.env`

Создай файл `.env` в корне проекта со следующим содержимым:

```
BOT_TOKEN=токен_от_BotFather
OWNER_ID=твой_telegram_id
```

### 3. Запустить через docker-compose

```bash
# Собрать и запустить контейнер
docker-compose up -d --build

# Просмотреть логи
docker-compose logs -f

# Остановить контейнер
docker-compose down
```

### 4. Обновление

При изменениях в коде:

```bash
# Пересобрать контейнер
docker-compose build

# Перезапустить
docker-compose up -d
```

### Примечания

- Данные бота (`seen_ids.json`, `subscribers.json`, `seen_favourites.json`) хранятся в папке `./data` на хост-машине и монтируются в контейнер.
- Для работы в продакшене достаточно только `requirements.txt` — тестовые зависимости (`requirements-dev.txt`) не включаются в рабочий образ.

## Конфигурация

Основные параметры в начале `main.py`:

| Константа | По умолчанию | Описание |
|---|---|---|
| `BOT_TOKEN` | из переменной окружения | Токен Telegram-бота от BotFather |
| `OWNER_ID` | из переменной окружения | Telegram ID владельца для админ-команд и уведомлений об ошибках |
| `SHIKI_USER` | `WNR` | Ник на Shikimori |
| `DISPLAY_NAME` | `Ворга` | Отображаемое имя в сообщениях |
| `SHIKI_BASE_URL` | `https://shikimori.io` | Домен — менять при смене зеркала |
| `CHECK_INTERVAL` | `900` (15 мин) | Интервал проверки в секундах |
| `ERROR_NOTIFY_INTERVAL` | `1800` (30 мин) | Минимальная пауза между уведомлениями владельцу об ошибках |
| `DATA_DIR` | `.` | Папка для файлов состояния; задаётся через переменную окружения |

## Поведение при ошибках

Фоновый цикл проверки истории и избранного не останавливается из-за единичной ошибки Shikimori API, сети, парсинга или отправки сообщения. Бот:

- пишет полный traceback в лог;
- пробует отправить владельцу короткое сообщение об ошибке в Telegram;
- не отправляет такие уведомления чаще одного раза в `ERROR_NOTIFY_INTERVAL`;
- продолжает работу и повторяет проверку после `CHECK_INTERVAL`.

Если процесс Python полностью завершился, сервер выключился или Telegram недоступен, уведомление отправить не получится — в таких случаях нужны логи хостинга или внешний мониторинг.

## Файлы данных

Бот хранит три файла с состоянием — **не коммитить**:

- `seen_ids.json` — ID уже обработанных событий истории
- `subscribers.json` — список подписчиков
- `seen_favourites.json` — ID уже обработанных записей избранного

По умолчанию файлы создаются в текущей рабочей директории. Если нужно хранить их в другом месте (например, в папке постоянного хранилища на хостинге), укажи путь через переменную окружения:

```
DATA_DIR=/путь/к/папке
```

При первом запуске бот молча запоминает текущую историю и избранное, не спамит прошлыми событиями — отслеживает только новые.

## Перенос подписчиков при смене хостинга

1. Напиши боту `/export` — он пришлёт `subscribers.json` в чат
2. На новом хостинге запусти бота
3. Отправь боту файл с подписью `/import` в поле caption

## Рассылка сообщений подписчикам

1. Напиши боту `/broadcast`
2. Пришли сообщение для рассылки — поддерживаются текст, фото, видео, GIF, стикер, документ, голосовое
3. Бот покажет превью и предложит две кнопки: **📢 Отправить** или **❌ Отмена**
4. Если передумал до отправки превью — напиши `/cancel`

## Разработка и тестирование

### Установка зависимостей разработки

Для запуска тестов установи зависимости:

```bash
pip install -r requirements.txt -r requirements-dev.txt
```

### Запуск тестов

```bash
pytest tests/                        # Запустить все тесты
pytest tests/test_notify.py -v       # Конкретный файл с подробным выводом
pytest tests/ -v --cov               # С измерением покрытия кода
```

**Важно:** Тесты требуют переменные окружения `BOT_TOKEN` и `OWNER_ID` (они автоматически устанавливаются в `conftest.py` для тестирования).
